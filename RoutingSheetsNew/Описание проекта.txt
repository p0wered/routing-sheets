================================================================================
                         ROUTING SHEETS API
              Система формирования маршрутных листов
================================================================================

ОБЩЕЕ ОПИСАНИЕ
--------------
Проект представляет собой серверное Web API приложение для автоматизации 
процесса формирования и управления маршрутными листами на производственном 
предприятии.

Маршрутный лист — это документ, содержащий последовательность операций, 
необходимых для изготовления детали или сборки изделия.


ПРЕДМЕТНАЯ ОБЛАСТЬ
------------------
Система предназначена для использования на заводе и охватывает следующие 
бизнес-процессы:

• Планирование производства — управление позициями плана производства, 
  которые определяют какие изделия и в каком количестве необходимо произвести.

• Формирование маршрутных листов — создание документов, описывающих 
  последовательность технологических операций для изготовления изделий.

• Управление операциями — добавление, редактирование и удаление операций 
  в маршрутных листах, назначение исполнителей.

• Отслеживание статусов — контроль состояния маршрутных листов и операций 
  на всех этапах производственного процесса.

Язык:               C# 12


АРХИТЕКТУРА ПРИЛОЖЕНИЯ
----------------------
Приложение построено по принципу монолитной архитектуры с разделением на слои:

1. Presentation Layer (Controllers)
   - Обработка HTTP-запросов
   - Валидация входных данных
   - Формирование ответов

2. Data Access Layer (Entity Framework)
   - Работа с базой данных
   - Определение моделей данных
   - Конфигурация связей между сущностями

3. DTOs (Data Transfer Objects)
   - Объекты для передачи данных между клиентом и сервером
   - Отделение внутренних моделей от внешнего API

МОДЕЛЬ ДАННЫХ
-------------
Система содержит следующие основные сущности:

МАРШРУТНЫЙ ЛИСТ (RoutingSheet)
  Центральная сущность системы. Содержит информацию о документе:
  номер, наименование, связь с позицией плана и изделием, статус, количество.

ОПЕРАЦИЯ (Operation)
  Технологическая операция в составе маршрутного листа. Включает:
  порядковый номер, код, наименование, цех, тип операции, исполнитель,
  цена, сумма, количество.

ПОЗИЦИЯ ПЛАНА ПРОИЗВОДСТВА (PlanPosition)
  Элемент производственного плана: код позиции, наименование, 
  связь с изделием, планируемое количество.

ИЗДЕЛИЕ (ProductItem)
  Продукция предприятия: наименование, описание, планируемое количество.

ЦЕХ (Guild)
  Производственное подразделение, в котором выполняются операции.

ТИП ОПЕРАЦИИ (OperationType)
  Классификатор видов технологических операций 
  (сборка, сварка, токарная обработка и т.д.).

ИСПОЛНИТЕЛЬ (Performer)
  Работник, назначенный на выполнение операции: ФИО, должность.

ЕДИНИЦА ИЗМЕРЕНИЯ (Unit)
  Справочник единиц измерения (шт., кг, м и т.д.).

СТАТУС МАРШРУТНОГО ЛИСТА (RoutingSheetStatus)
  Справочник статусов: Черновик, Активен, Завершен, Отменен.

СТАТУС ОПЕРАЦИИ (OperationStatus)
  Справочник статусов: Ожидает, В работе, Завершена, Отменена.


СВЯЗИ МЕЖДУ СУЩНОСТЯМИ
----------------------
• Маршрутный лист → Позиция плана (многие к одному)
• Маршрутный лист → Изделие (многие к одному)
• Маршрутный лист → Единица измерения (многие к одному)
• Маршрутный лист → Статус МЛ (многие к одному)
• Маршрутный лист → Операции (один ко многим)

• Операция → Маршрутный лист (многие к одному)
• Операция → Статус операции (многие к одному)
• Операция → Цех (многие к одному)
• Операция → Тип операции (многие к одному)
• Операция → Исполнитель (многие к одному)

• Позиция плана → Изделие (многие к одному)


ФУНКЦИОНАЛЬНЫЕ ВОЗМОЖНОСТИ
--------------------------
1. Справочники (только чтение):
   - Статусы маршрутных листов
   - Статусы операций
   - Изделия

2. Справочники (полный CRUD):
   - Единицы измерения
   - Цеха
   - Типы операций
   - Исполнители
   - Позиции плана производства

3. Маршрутные листы:
   - Просмотр списка с фильтрацией по позиции плана и изделию
   - Просмотр детальной информации с операциями
   - Создание, редактирование, удаление
   - Изменение статуса
   - Разбиение на несколько листов с переносом операций

4. Операции:
   - Просмотр списка с фильтрацией по маршрутному листу и цеху
   - Создание, редактирование, удаление
   - Назначение и снятие исполнителя
   - Изменение статуса


ОСОБЕННОСТИ РЕАЛИЗАЦИИ
----------------------
• Каскадное удаление операций при удалении маршрутного листа
• Автоматическая перенумерация операций при удалении
• Уникальность номера маршрутного листа
• Предзаполненные справочники статусов (seed data)
• Тестовые данные для быстрого старта
• Валидация внешних ключей при создании/обновлении
• Поддержка JSON с обработкой циклических ссылок
