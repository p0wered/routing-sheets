## 1. Обзор проекта

Проект представляет собой веб-приложение (админ-панель) для управления маршрутными листами на производственном предприятии. Состоит из двух частей:

- **Backend** — ASP.NET Core Web API (.NET 8.0), обеспечивает REST API, JWT-авторизацию и работу с базой данных MS SQL Server.
- **Frontend** — React SPA (Vite + TypeScript + Tailwind CSS), предоставляет пользовательский интерфейс для работы с системой.

### Предметная область

Маршрутный лист — документ, описывающий последовательность технологических операций для изготовления изделия. Система автоматизирует:

- Планирование производства (позиции плана, изделия)
- Формирование и управление маршрутными листами
- Управление операциями (добавление, назначение исполнителей, смена статусов)
- Ведение справочников (цеха, типы операций, исполнители, единицы измерения)

### Взаимодействие компонентов

1. Пользователь открывает React SPA в браузере
2. При входе отправляется `POST /api/auth/login` с логином и паролем
3. Backend проверяет учётные данные, возвращает JWT-токен
4. Токен сохраняется в `localStorage` и подставляется в заголовок `Authorization: Bearer <token>` для каждого запроса
5. Backend валидирует токен через JWT middleware и обрабатывает запрос
6. При ответе 401 фронтенд автоматически перенаправляет на страницу логина

## 5. Система авторизации

### Модель пользователя (User)

| Поле         | Тип     | Описание                         |
|--------------|---------|----------------------------------|
| Id           | int     | Первичный ключ                   |
| Username     | string  | Логин (уникальный, макс. 100)    |
| PasswordHash | string  | BCrypt-хэш пароля                |
| FullName     | string  | ФИО (макс. 300)                  |
| Role         | string  | Роль (макс. 50)                  |

### Процесс аутентификации

1. Клиент отправляет `POST /api/auth/login` с `{ username, password }`
2. Сервер ищет пользователя по `username`
3. Проверяет пароль через `BCrypt.Verify()`
4. Генерирует JWT-токен со следующими claims:
   - `NameIdentifier` — ID пользователя
   - `Name` — имя пользователя (username)
   - `GivenName` — ФИО
   - `Role` — роль
5. Возвращает `{ token, user: { id, username, fullName, role } }`

### Конфигурация JWT

| Параметр      | Значение                              |
|---------------|---------------------------------------|
| Issuer        | RoutingSheetsAPI                      |
| Audience      | RoutingSheetsClient                   |
| Алгоритм      | HMAC-SHA256                           |
| Время жизни   | 480 минут (8 часов — рабочий день)    |
| Хранение      | `localStorage` на клиенте             |

### Фронтенд: AuthContext

- Хранит текущего пользователя и токен
- При загрузке приложения проверяет токен вызовом `GET /api/auth/me`
- При ответе 401 автоматически очищает данные и перенаправляет на `/login`
- Предоставляет функции `login()` и `logout()` через React Context

## 10. Запуск проекта

### Backend

```bash
cd backend/backend
dotnet run
```

API будет доступно по адресу: `https://localhost:7001`
Swagger UI: `https://localhost:7001/swagger`

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Фронтенд будет доступен по адресу: `http://localhost:5173`

> Vite проксирует все запросы `/api/*` на `https://localhost:7001`, поэтому для разработки достаточно запустить оба сервера.

### Порядок запуска

1. Убедитесь, что MS SQL Server запущен
2. Запустите бэкенд (`dotnet run`) — при первом запуске автоматически создастся БД, применятся миграции и добавится тестовый пользователь
3. Запустите фронтенд (`npm run dev`)
4. Откройте `http://localhost:5173` в браузере
5. Войдите с логином `chief` и паролем `chief123`
